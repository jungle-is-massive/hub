01
Data sources

The dashboard pulls from three systems. Each one owns a specific part of the picture — they are not duplicates of each other.

HubSpot
CRM
The primary source for contacts, lifecycle stages, and deals. The dashboard queries HubSpot via a Supabase edge function proxy (hs-proxy) to avoid CORS issues. Two object types are used: Contacts (for Prospect Funnel Stages) and Deals (for Deal Opportunity Tracking). All deal queries are scoped to the Agency Pipeline only.
Synergist
Project System
Jungle's internal project management system. Used for the Open Deals stat card — specifically the "live" count of active opportunities being handled by Kitty Lewis-Williams, Jade Raad, or Coris Leachman. Queried via the same hs-proxy edge function using a separate Synergist target.
Supabase
Database
Jungle's own database. Used to store the funnel stage definitions — the list of boxes, their labels, order, and metadata (e.g. which is MQL, which is SQL). This means the funnel structure can be updated without touching code. Supabase also stores contacts imported from Dripify.

03
Deal Opportunity Tracking

The five boxes immediately below the stat cards. These count deals in HubSpot — specifically in the Agency Pipeline. Each box maps to one or more deal stages in that pipeline.

Opportunities and Pitches pull from the active deals already loaded for the stat cards. Won and Lost run separate queries because closed stages are excluded from the main active deals fetch.

Agency Pipeline only. There are four pipelines in HubSpot (Jungle Sales Pipeline, Agency Pipeline, Partnerships, IP Sponsorship). This section deliberately shows Agency Pipeline only — the one Coris is actively managing BD in. The others are not counted here.

04
Prospect Funnel Stages

The six boxes in the bottom row. These count contacts in HubSpot by their lifecycle stage — not deals. A contact's lifecycle stage is a separate property to what pipeline their deal sits in.

The stage structure is defined in Supabase (the funnel_stages table) and the counts are fetched live from HubSpot on each page load, one request per stage in parallel.

ℹ️
Contacts, not deals. A contact tagged as Opportunity lifecycle stage (45 currently) is not the same as 45 deals in the Opportunity deal stage. Lifecycle stage tracks where a person is in the relationship — deals track active commercial conversations. Both matter, but they answer different questions.

05
Known gaps & next steps

Things that are intentionally deferred, pending a decision, or blocked on HubSpot configuration access.

  • Won (box 09) always shows — — Agency Pipeline has no Closed Won stage. Options: add one in HubSpot (Settings → Objects → Deals → Pipelines → Agency Pipeline → Add stage), or map Won to pull from a different pipeline. Needs a decision on which is right for the business.
  • Declined (box 11) always shows — — No equivalent stage exists in HubSpot yet. To be added to Agency Pipeline when the team is ready to track this separately from Lost.
  • Nurturing (box 06) always shows — — HubSpot's lifecycle stage list doesn't have a Nurturing option by default. Can be added as a custom lifecycle stage in HubSpot (Settings → Objects → Contacts → Lifecycle Stage). Confirmed that HubSpot does allow custom lifecycle stages.
  • SQL (box 03) shows 0 — There are currently zero contacts tagged as Sales Qualified Lead in HubSpot. This is accurate data, not a bug — SQL criteria and the process for moving contacts from MQL to SQL hasn't been defined yet.
  • Jungle Sales Pipeline not currently tracked — There are 21 deals in the Jungle Sales Pipeline (the default pipeline) which are not shown anywhere on this dashboard. This was a deliberate decision as of April 2026 — Coris is focused on Agency Pipeline. To be revisited once there is clarity on which pipeline the team considers primary.
  • Data source documentation per box — A slide-in panel explaining the source for each box is planned for the dashboard. Deferred until the numbers are confirmed as correct and stable.
